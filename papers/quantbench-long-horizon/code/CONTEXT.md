## Key Financial Terms / 关键金融术语

| Term (EN) | Term (CN) | Definition |
|-----------|-----------|------------|
| Market Regime | 市场制度 | A persistent market state characterized by distinct statistical properties (volatility, drift, correlation). Types: Bull, Bear, Crash, Recovery, Stagnation. |
| Regime Transition | 制度转换 | The change from one market regime to another. The most dangerous event for quantitative strategies. |
| Survival Rate | 存活率 | Probability of maintaining capital above a ruin threshold ($10M from $100M start). Key long-horizon metric. |
| Recovery Time | 恢复时间 | Trading days from maximum drawdown trough to next equity high-water mark. |
| CAGR (Compound Annual Growth Rate) | 复合年增长率 | Annualized return over the full simulation horizon. |
| Maximum Drawdown | 最大回撤 | Largest peak-to-trough decline in portfolio value. |
| Sharpe Ratio | 夏普比率 | Risk-adjusted return: (annualized_return - risk_free_rate) / annualized_volatility. |
| Momentum Strategy | 动量策略 | Overweights recently outperforming assets, underweights underperformers. Vulnerable to momentum crashes. |
| Mean Reversion | 均值回归 | Overweights recently underperforming assets expecting price normalization. Vulnerable in trending markets. |
| Carry Strategy | 套息策略 | Favors assets with positive yield or roll return. Vulnerable in high-volatility regimes. |
| Volatility Targeting | 波动率目标 | Adjusts position sizes to maintain target portfolio volatility. |
| Tail Risk Hedging | 尾部风险对冲 | Protection against extreme losses via put options or other instruments. |
| Semi-Markov Process | 半马尔可夫过程 | Stochastic process where regime durations are random, calibrated to historical transition frequencies. |
| Transition Matrix | 转移矩阵 | Probability matrix governing regime-to-regime transitions. Calibrated to 1950-2024 US data. |

## Key ML Terms / 关键机器学习术语

| Term (EN) | Term (CN) | Definition |
|-----------|-----------|------------|
| Regime Detection | 制度检测 | Rule-based classifier using volatility, momentum, and correlation to classify current market state. |
| Detection Delay | 检测延迟 | Number of days between regime transition occurring and agent correctly identifying it. |
| Strategy Switching | 策略切换 | Agent's ability to change active trading strategy based on detected regime. |
| Adaptation Score | 适应度分数 | Composite metric combining strategy change frequency with effectiveness of each change. |
| Ensemble Agent | 集成Agent | Multi-strategy agent combining signals from momentum, mean-reversion, and regime-aware sub-agents. |
| Risk Manager | 风险管理器 | Module monitoring drawdown, volatility, and exposure, applying automatic de-risking when limits are breached. |
| Lookback Window | 回望窗口 | Number of historical days used for computing signals (momentum=60, mean-reversion=20). |

## Variable Naming Conventions / 变量命名约定

- `N`: Total number of assets in the simulated universe
- `num_days`: Simulation length in trading days (default 1008 for quick tests, 2520 for full 10-year)
- `initial_capital=1e8`: Starting capital ($100M)
- `ruin_threshold=1e7`: Survival threshold ($10M, 90% loss)
- `transaction_cost_bps=5.0`: Per-trade cost in basis points
- `slippage_bps=2.0`: Execution slippage in basis points
- `target_vol`: Target annualized portfolio volatility for vol-targeting
- `max_drawdown_trigger`: Drawdown level triggering automatic de-risking
- `lookback_short=10`, `lookback_medium=30`, `lookback_long=60`: Regime detection windows

## Data Format Descriptions / 数据格式说明

- **Market Data DataFrame**: Columns include `day`, `regime`, `is_transition`, plus `{asset_name}_return` for each asset. Generated by MarketSimulator.simulate_scenario().
- **ScenarioConfig**: Defines scenario name, duration, regime sequence (optional), custom transition matrix, and random seed.
- **AgentResult**: Complete evaluation output: final_capital, CAGR, max_drawdown, sharpe, survival, recovery_days, regime_detection_accuracy, equity_curve.
- **RegimeParams**: Per-regime parameters: drift_annual, vol_annual, bond_drift, commodity_drift, base_correlation, spread_multiplier, duration_range.
- **TradeAction**: Agent's decision: target_weights (ndarray), strategy (enum), risk_limit, hedge_ratio.

## 中文补充说明

本文代码实现了QuantBench基准框架：(1) 使用半马尔可夫过程生成具有制度转换的多资产市场数据，(2) 提供四种交易Agent（动量、均值回归、制度感知、集成），(3) 在长周期模拟中评估Agent的存活率、恢复时间、制度检测准确率等指标。关键设计决策包括使用基于规则的制度检测器（短期波动+中期动量+长期相关性），风险管理器在回撤触发时自动减半仓位，以及使用历史制度序列（如2008金融危机：牛→熊→崩盘→复苏→牛）进行场景校准。多资产宇宙包含股票、债券、商品、货币和加密货币，各类资产在不同制度下具有不同的漂移率和波动率参数。
